05 / Full Article Text (agent-readable)

I took a simple agent I actually use, built it on three different platforms, and paid attention to what actually holds up when it needs to run every week without drama. The agent: Takes a newsletter called FOMA (thanks, Adir Duchan (ההוא מהזה) !) and summarizes it in an internal format for Monday employees, then sends it to relevant Slack channels. On paper, it’s simple: Weekly trigger → fetch email → summarize → send. That’s exactly where the differences between tools come into play. First — what kind of agents are we even talking about? Agent’s Wave’s There’s a lot of confusion in the industry, so here’s the framework that makes sense to me: Wave 1: Chat. ChatGPT and friends. Give a prompt, get an answer. No process, no continuity. Wave 2: Agentic Flow. n8n, Cursor, Claude Code. You connect models to real tools; there is memory involved, but you’re still the one planning everything. Wave 3: Autonomy (sort of). Claude CoWork, OpenClaw. You give a goal, and the system figures out how to reach it. It’s not fully there yet, but it’s close enough to be genuinely interesting. n8n: It works. Period. But you’ll earn it. I built a classic flow: Weekly schedule (with means there is a cron job working) → Gmail search for FOMA → grab latest email → send to model → format → Slack You can notice that the agent is only part of the flow, the schedule, and the output formatting, as well as sending an email are deterministic steps that have to happen. Here is the flow: n8n agentic flow What’s good: It runs smoothly. When something breaks, you know exactly where it is. Full control. What’s not: Every small thing needs configuration. You have to think through everything up front. Setup time is real. You can reduce it by using n8n MCP with Cursor or Claude Code, for example, and then you’ll get to the result very fast, but you’ll still need to configure the steps. Claude CoWork: Feels like magic. Until it doesn’t. Here I did something different. I just wrote: “Every week, find the latest FOMA newsletter, summarize it, and send it to relevant Slack channels.” And it… did it. Connected Gmail on its own. Understood what FOMA was. Handled messy, unstructured text. Sent to Slack. No flow. No nodes. No thinking required on my end. That’s genuinely impressive. The agent’s choices and steps But here’s the honest part: It’s a very new tool, and I find it hard to understand how it works and when it works. I lack visibility into the agent’s actions (not to talk about evaluation, which is a must for production-grade agents): Claude CoWork Agent The experience is amazing — but the control is almost nonexistent. It feels like: “I’ll say what I want, and it’ll figure it out.” Sometimes it does. Sometimes less so. OpenClaw: This isn’t a tool; it feels like a learning machine OpenClaw isn’t playing the same game. I didn’t build a flow, and I didn’t give it a single task. I did an onboarding: defined context, added tasks, and connected it to a board. And what happens is that it starts to understand — over time. The first step is the configuration of the Soul, Identity, and info about me, the owner (so much cringe in these specific names..): OpenClaw Configurations But there’s something different here: it’s not just executing, it’s developing. And that’s both the advantage and the problem. The first go at creating the task for FOMA news The current reality: setup isn’t smooth, integrations are limited, and it’s not production-ready. But when it works, it feels like the future. So, which one should you use? Honestly? I’m not sure yet. We’re only at the beginning of this third wave of agents, and we’re still figuring out how to actually leverage autonomous agents in practice. A definitive “use this, not that” would be premature, and anyone telling you otherwise is oversimplifying. My guess: tools from the second wave, like n8n, will either evolve toward full autonomy, or we’ll collectively realize that some use cases actually want deterministic steps and consciously choose to keep building those on second-wave tools. Both outcomes are valid. Both might happen at the same time. What I do know: this space is moving fast, and paying attention right now matters.
